硅谷二手Mac mini被抢疯了,背后推手竟是一只虚拟螃蟹

Wei Family LLC2026-01-272026-01-27

上周我刷推特的时候，看到一个很荒诞的现象：硅谷的二手Mac mini价格突然被炒高了。

我第一反应是苹果又发了什么新品？点进去一看,完全不是那回事。

推高价格的幕后黑手，是一个叫Clawdbot的开源项目——logo是一只红色的龙虾（或者螃蟹？）。更离谱的是，连Google AI Studio的产品经理都在推上”安利”它，还专门买了台Mac mini来跑这玩意儿。

一个AI项目，把硬件都带火了。这事儿够新鲜的。

我为什么会注意到它？

说实话，这两年AI产品我见太多了。ChatGPT、Claude、各种Agent框架……审美疲劳都不足以形容我的状态。

但Clawdbot引起我注意的原因很简单：它解决了一个真实的痛点。

我每天要处理大量重复性工作——清理邮件、整理日程、同步待办事项。Siri？不提了，到今天还是个智障。ChatGPT确实聪明，但它被困在浏览器的沙盒里，除了陪我聊天，什么也干不了。

而Clawdbot的定位完全不同：它是一个真正能”动手”的AI。

不是生成文字让你复制粘贴，而是直接操作你的电脑——打开应用、执行命令、读写文件、控制浏览器。

说白了，这是一个有”手”的AI。

它到底是什么？

简单讲，Clawdbot是一个可以部署在本地或云端的私人AI Agent。

它有两个核心组件：

  • Agent：负责思考和推理，可以调用Claude、GPT等大模型
  • Gateway：负责连接你的数字生活，打通Telegram、WhatsApp、Discord等即时通讯软件

关键区别在于：和那些被困在网页里的AI不同，Clawdbot运行在你的私有环境中，拥有系统级权限。

它可以：

  • 直接访问你的文件系统
  • 运行终端命令
  • 控制浏览器（打开网页、点击按钮、填写表单）
  • 通过Cron任务主动向你发送消息

最后一点是我觉得最颠覆的——它不是等你发指令才响应，而是会主动找你。

实际能干什么？（几个让我眼前一亮的案例）

说那么多不如看实操。

案例1：自动化开发工作流

有用户通过Telegram给Clawdbot发了一条消息：「帮我跑一下这个仓库的测试」

然后它自己完成了以下动作：

  1. 拉取代码仓库
  2. 打开VS Code
  3. 运行测试
  4. 如果失败，自动生成修复代码
  5. 测试通过后自动提交

整个过程用户在遛狗，手机只收到了一条通知：「搞定了」。

案例2：每日工作简报

一个用户设置了定时任务：每天早上9点，Clawdbot自动给他发一条Telegram消息，内容包括：

  • 今日日历安排（直接从macOS Calendar提取）
  • 待办任务（按优先级排序）
  • 需要注意的事项

用户的原话是：「感觉像有个助理在远程帮我整理工作」。

案例3：自己给自己写技能

这个最骚。

有人在对话里说：「我想自动化Todoist的一些任务，你能帮我搞吗？」

Clawdbot问了几个问题，搞清楚需求后，自己写了一个skill（技能插件），然后自己装好，开始用。

用户全程没碰过代码。

说说安装

这事儿必须得讲清楚，因为门槛确实存在。

如果你只是想体验核心功能，买个VPS部署Node.js环境就能跑。

但如果你想让它控制本地应用（比如iMessage、macOS日历、本地文件），就需要在Mac或PC上运行。

这就是为什么二手Mac mini被炒起来的原因——它功耗低、性能够、7x24小时开机刚刚好。

安装命令很简单：

一行代码，然后跟着向导走就行。

官方网站：https://clawd.bot

亮点有哪些？

1. 系统级权限 + 主动交互

目前市面上的AI大多是「被动响应式」——你问它才答。而且被困在网页沙盒里，除了生成文字，啥也干不了。

Clawdbot完全不同：

  • 它有Shell访问权，可以执行终端命令、读写文件、控制浏览器
  • 它通过Cron任务调度，能主动给你发简报、警报或建议

这是从”聊天机器人”到”真正的助理”的质变。

2. 持久化记忆

Siri和大多数大模型的痛点：上下文遗忘。

Clawdbot引入了持久化记忆层，把交互历史和用户偏好存储为Markdown文档或向量数据。

这意味着什么？

它会记得你两周前随口提到的旅行计划，并在临期时主动调取天气数据和航班信息提醒你。

3. 从对话到执行

有个典型案例是去SaaS化：

用户以前需要付费用Zapier来实现「RSS更新时自动创建Todoist任务」。

现在？让Clawdbot写一段Python脚本，通过API监听RSS源，自动创建任务。

省掉了一笔订阅费，还更灵活。

必须说的风险

给AI系统级权限是极其危险的事情。

这不是危言耸听。你本质上是给一个AI开了sudo权限——如果它”幻觉”了，可能会删掉你的重要文件。

Clawdbot提供了一些安全措施：

  • Docker沙盒模式：限制AI的破坏半径
  • 白名单机制：严格配置哪些用户可以通过Telegram/WhatsApp控制你的主机

我的建议是：如果你要用，一定要先在测试环境跑一段时间，别一上来就接生产环境。

当前的局限性

必须承认，Clawdbot还处于早期阶段。

几个现实问题：

  1. 技术门槛：需要会用命令行、申请API Key、有基本的服务器运维能力
  2. 稳定性：官方自己描述是”脆弱的”，可能因为上下文超长或API变动而报错
  3. 资源消耗：24小时运行意味着持续的API调用费用

它更适合开发者和极客，不是普通消费者产品。

我自己的体验

折腾了两天，我把它部署在家里的Mac mini上。

第一个让我惊艳的瞬间：我在Telegram里随口说了一句「明天9点提醒我给张总打电话」，它回复「收到」。

第二天早上8:55，我收到了提醒。不是手机自带的那种冷冰冰的闹钟，而是它发了条消息：「老板，9点要给张总打电话，要不要我帮你查一下他上次聊的内容？」

这种”有温度”的主动交互，是传统助理类App从来没给过我的体验。

当然，它也翻过车——有次我让它整理下载文件夹，它误删了一些我还没看的PDF。

但这恰好说明：它真的在”干活”，不是在演戏。

写在最后

Clawdbot让我看到了一种可能性：后App时代的交互逻辑。

未来我们可能不再频繁打开几十个App，而是通过一个拥有系统级权限的AI接口，调度所有底层服务。

有人把它叫做”AI操作系统的雏形”，我觉得有点道理。

如果你有一定的技术背景，想体验完全掌控数据的AI助手，Clawdbot目前是最接近《钢铁侠》贾维斯的开源原型。

但如果你只是想要一个开箱即用的工具，建议再等等——等到社区把边边角角打磨好，再入坑不迟。
